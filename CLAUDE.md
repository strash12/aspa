# LLM Wiki Agent — АСПА

## Роль
Ты — агент базы знаний проекта АСПА (OTFS FRFT-SIC channel estimation).
Твоя задача — поддерживать организованную wiki из markdown-файлов,
строить связи между концепциями, и отвечать на вопросы по проекту.

## Архитектура

```
aspa/
├── raw/           # Сырые источники (статьи, PDF, транскрипты)
├── wiki/          # Организованная база знаний
│   ├── index.md   # Индекс всех wiki-страниц
│   ├── log.md     # История операций ingest/lint
│   └── *.md       # Wiki-страницы с backlink-ами
├── position_1/    # Положение 1: формирование кадра
├── position_2/    # Положение 2: канал
├── position_3/    # Положение 3: FRFT-SIC
├── documentation/ # Литература, QuaDRiGa source, tutorials
├── AI_CONTEXT.md  # Полный контекст проекта
├── activity.md    # Лог задач
└── hot_cache.md   # Последние 5 действий
```

## Правила ingest (приём нового контента)

1. Прочитать файл из `raw/`
2. Извлечь ключевые концепции, формулы, методы, ссылки
3. Создать/обновить wiki-страницы в `wiki/`
4. Обновить `wiki/index.md` — добавить новые страницы и связи
5. Обновить `wiki/log.md` — записать операцию
6. Если есть backlink-и — обновить связанные страницы

## Формат wiki-страницы

```markdown
# Название

## Описание
Краткое описание (2-3 предложения)

## Связи
- [[Другая страница]] — описание связи
- [[Ещё одна]] — описание связи

## Теги
#тег1 #тег2 #тег3

## Содержание
Основной контент...

## Источники
- [[raw/имя_файла.md]]
```

## Правила lint (проверка целостности)

1. Проверить что все backlink-и в `index.md` ведут к существующим файлам
2. Найти страницы без связей (orphan) — предложить связи
3. Найти дубликаты или сильно пересекающиеся страницы
4. Проверить что `index.md` актуален
5. Записать результаты в `wiki/log.md`

## Когда использовать wiki vs другие файлы

- **wiki/** — концепции, методы, формулы, связи между идеями
- **AI_CONTEXT.md** — текущий статус проекта, архитектура, ключевые решения
- **activity.md** — лог начатых/завершённых задач
- **hot_cache.md** — последние 5 действий (текущий статус)
- **position_*/** — код и документация конкретных положений

## Правило git

При каждом значительном изменении файлов проекта (создание новых файлов, изменение архитектуры, завершение задачи):
1. `git add` — добавить изменённые файлы по имени (НЕ `git add -A`)
2. `git commit` — создать коммит с понятным сообщением
3. `git push` — отправить в remote

**Не коммитить:** `.env`, секреты, временные файлы, `hot_cache.md`

## При запросе вопроса по проекту

1. Прочитать `wiki/index.md` для навигации
2. Перейти по relevant backlink-ам к нужным wiki-страницам
3. Если нужно — прочитать `AI_CONTEXT.md` для текущего статуса
4. Ответить на основе wiki, указав источники
