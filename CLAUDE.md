## При начале каждой сессии

1. **Проверить наличие `AI_CONTEXT.md`** в корне текущего проекта.
   Если файл существует — прочитать его полностью перед началом работы.

3. **Проверить наличие `hot_cache.md`** в рабочей директории проекта.
   Если файл существует — прочитать последние записи для понимания контекста текущей работы.

## Обязательная документация проекта

**Каждый проект должен содержать три файла в корне рабочей директории:**

| Файл | Назначение | Когда создавать |
| --- | --- | --- |
| `AI_CONTEXT.md` | Полный контекст: архитектура, структура файлов, ключевые решения, API, статус | При создании первого файла проекта |
| `activity.md` | Лог задач: начатые `[START ...]` и завершённые `[DONE ...]` задачи | При начале любой новой задачи |
| `hot_cache.md` | Последние 5 действий с timestamp и статусом | При каждом изменении файлов |

**Почему это обязательно:**

- Сессии прерываются — контекст теряется, документация позволяет продолжить с места остановки
- Работу могут вести разные агенты — каждый должен понимать что сделано и что делать дальше
- Без документации каждый новый запуск начинается «с чистого листа» — теряется время и контекст
- `AI_CONTEXT.md` — это «память» проекта, `activity.md` — «план работ», `hot_cache.md` — «текущий статус»

**Правила ведения:**

1. **При создании первого файла проекта** — сразу создать `AI_CONTEXT.md`, `activity.md`, `hot_cache.md`
2. **При начале новой задачи** — добавить запись в `activity.md` с `[START дата] описание`
3. **При завершении задачи** — заменить `[START ...]` на `[DONE дата]` в `activity.md`
4. **При каждом изменении файлов** — обновить `hot_cache.md` (макс. 5 записей, oldest удаляется)
5. **При изменении архитектуры или ключевых решений** — обновить `AI_CONTEXT.md`

## Правило ведения hot_cache.md

При каждом изменении файлов проекта (создание, редактирование, удаление):
1. Обновить `hot_cache.md` в рабочей директории проекта — добавить новую запись с:
   - Timestamp в формате `YYYY-MM-DD HH:MM`
   - Статус: `завершено`, `в работе`, `ожидает`, `отменено`
   - Краткое описание задачи
   - Список изменённых файлов
2. Хранить не более 5 записей — при добавлении новой удалять oldest
3. При изменении статуса существующей задачи — обновить timestamp и статус
4. Файл `hot_cache.md` должен пересекаться с другой документацией проекта — указывать связанные файлы
