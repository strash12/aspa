# Положение 1 — Формирование OTFS кадра

## Назначение
Описание процесса формирования OTFS кадра: от битового потока до временного сигнала.
Этот документ служит основой для положений 2 (прохождение через канал) и 3 (оценка канала FRFT-SIC).

---

## 1. Общая структура OTFS системы

```
Биты → QPSK → DD-сетка (M×N) → Вставка пилота → ISFFT → Heisenberg → CP/ZP → Время-сигнал
```

OTFS модулирует символы в **Delay-Doppler (DD)** области, а не во времени или частоте.
Ключевое преимущество: DD-канал почти статичен даже при высоких скоростях,
в отличие OFDM где канал быстро меняется во времени.

---

## 2. Параметры DD-сетки (из положения 3)

| Параметр | Значение | Описание |
|---|---|---|
| M | 64 | Число поднесущих (размер по задержке) |
| N | 32 | Число DD-символов (размер по Допплеру) |
| Δf | 150 кГц | Межподнесущий интервал |
| T | 1/Δf ≈ 6.67 мкс | Длительность одного поднесущего символа |
| T_otfs = N·T | ≈ 213.3 мкс | Полная длительность OTFS кадра |
| B = M·Δf | 9.6 МГц | Полоса |
| fc | 5.9 ГГц | Несущая частота |
| Модуляция | QPSK (mod_size=4) | 2 бита на символ |

### Разрешающая способность DD-сетки

```
Δτ = 1/(M·Δf) = 1/9.6e6 ≈ 104 нс     — разрешение по задержке
Δν = 1/(N·T)   = 1/213.3e-6 ≈ 4.69 кГц — разрешение по Допплеру
```

При скорости v=80 м/с (≈288 км/ч) на fc=5.9 ГГц:
```
f_Doppler = fc·v/c = 5.9e9·80/3e8 ≈ 1573 Гц
ν̃ = f_Doppler / Δν ≈ 1573/4687.5 ≈ 0.335 bins  (в пределах одного бина)
```

Но при df=15 кГц (мотивирующий пример): ν̃ ≈ 1.66 bins — **дробный Допплер**,
символы "размазываются" по нескольким бинам DD-сетки.

## 3. Структура DD-кадра с встроенным пилотом

### 3.1 Расположение пилота

Пилот размещается в центре DD-сетки:
```
k_p = N/2 = 16    — индекс по Допплеру
l_p = M/2 = 32    — индекс по задержке
```

### 3.2 Guard-зона

Вокруг пилота создаётся защитная зона (нулевые символы) для предотвращения
интерференции пилота с данными из-за дробного Допплера и многолучевости:

```
L_guard_delay = 14    — guard по задержке (±14 bins от пилота)
L_guard_dopp  = 10    — guard по Допплеру (±10 bins от пилота)
```

Размер guard-зоны:
```
По задержке:  [l_p - L_guard_delay, l_p + L_guard_delay] = [18, 46]
По Допплеру:  [k_p - L_guard_dopp,  k_p + L_guard_dopp]  = [6, 26]
```

### 3.3 Амплитуда пилота

```
pilot_power = 100
x_p = sqrt(pilot_power) = 10
```

Пилот значительно мощнее данных (QPSK символы имеют единичную амплитуду),
что обеспечивает хорошее отношение сигнал/шум для оценки канала.

### 3.4 Визуальная структура DD-кадра

```
        Doppler (N=32)
        ^
    k=31|  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  |
     ...|  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  D  |

        l=0                                     Delay (M=64)                                    l=63

D = данные (QPSK)
G = guard (нули)
P = пилот
```

### 3.5 Число символов данных

```
Всего bins:          M × N = 64 × 32 = 2048
Пилот:               1 bin
Guard bins:          ≈ (2·L_guard_delay+1) × (2·L_guard_dopp+1) - 1
                     = 29 × 21 - 1 = 608 bins
Data bins:           2048 - 1 - 608 = 1439 bins
Data bits (QPSK):    1439 × 2 = 2878 бит на OTFS кадр
```

## 4. Процесс модуляции (TX chain)

### 4.1 DD-сетка → частотно-временная область (ISFFT)

Символы из DD-области [M×N] преобразуются в частотно-временную область через
**2D обратное преобразование Фурье** (аналог 2D-IFFT):

```
X_tf(n, m) = (1/√(MN)) · Σ_k Σ_l X_dd(k, l) · e^(j2π(nk/N - ml/M))
```

В MATLAB: `X_tf = ifft2(X_dd.')`

### 4.2 Частотно-временная область → время (Heisenberg / OFDM)

Каждая строка X_tf обрабатывается через IFFT по поднесущим (M точек),
затем добавляется циклический префикс или нулевой префикс:

```
x_time(t) = Σ_m X_tf(n, m) · e^(j2πmΔf·t) · rect(t/T)
```

В MATLAB: `x_time = ifft(X_tf, [], 2)` → reshape → добавить CP/ZP

### 4.3 Добавление префикса

В положении 3 используется **Zero Padding** (padType='ZP'):
```
padLen = 12    — длина нулевого префикса (сэмплов)
```

ZP предпочтительнее CP для OTFS, т.к. упрощает анализ свёртки в DD-области.

---

## 5. Матричная модель передачи

Полный процесс модуляции можно записать как линейное преобразование:

```
s = F_OTFS · x_dd
```

где:
- `x_dd` — вектор символов DD-области (M×N × 1)
- `F_OTFS` — матрица OTFS-модуляции (размерность N_s × N_s, N_s = MN + padLen)
- `s` — временной сигнал

## 6. Связь с другими положениями

### → Положение 2 (Канал)
Сформированный кадр проходит через многолучевой канал QuaDRiGa (3GPP 38.901 UMi NLOS):
```
r = G · s + w
```
где G — матрица канала, w — шум.

Канал вносит:
- Задержки (сдвиг по l в DD)
- Допплер (сдвиг по k в DD)
- Дробные сдвиги → "размазывание" по нескольким бинам

### → Положение 3 (Оценка канала FRFT-SIC)
На приёмной стороне после демодуляции (обратный процесс):
```
Hdd = demod(r)    → DD-отклик канала
```
Алгоритм FRFT-SIC оценивает параметры путей (задержка, Допплер, усиление)
из Hdd, используя:
1. Пилот в известной позиции (k_p=16, l_p=32)
2. Guard-зону для изоляции пилота от данных
3. FRFT для точной оценки дробных параметров
